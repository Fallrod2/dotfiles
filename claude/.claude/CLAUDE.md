# Préférences globales

## Style
- Réponses concises, pas de résumé de ce que tu viens de faire sauf demande
- Français ou anglais selon la langue de mon message
- Commits en anglais, format conventional commits (feat:, fix:, chore:...)

## Workflow
- Toujours donner la commande pour vérifier un changement (build/test)
- Ne jamais commit/push sans demande explicite
- Préférer éditer les fichiers existants plutôt qu'en créer de nouveaux

## Environnement
- macOS headless (Mac mini, accès SSH/tmux), éditeur Neovim (config mvim)
- Dotfiles gérés par GNU Stow dans ~/dotfiles
